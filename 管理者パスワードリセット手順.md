# 管理者パスワードリセット手順

## 現在の状況
- デフォルト値 `admin123` で認証に失敗
- Netlifyの環境変数 `ADMIN_PASSWORD` に別のパスワードが設定されている可能性

## 解決方法：パスワードをリセット

### ステップ1: Netlifyにログイン
1. https://app.netlify.com にアクセス
2. ログイン

### ステップ2: プロジェクトを選択
1. 左サイドバーの「**Projects**」をクリック
2. `kyushoku.site` のプロジェクトを選択

### ステップ3: 環境変数を確認・変更
1. 左サイドバーの「**Site settings**」（歯車アイコン⚙️）をクリック
2. 「**Environment variables**」を選択
3. `ADMIN_PASSWORD` を探す

#### パターンA: `ADMIN_PASSWORD` が存在する場合
- 現在の値を確認（目のアイコンをクリックして表示）
- または、新しいパスワードに変更：
  1. `ADMIN_PASSWORD` の行の右側の「**Edit**」をクリック
  2. 新しいパスワードを入力（例: `Kyushoku2024!Secure`）
  3. 「**Save**」をクリック

#### パターンB: `ADMIN_PASSWORD` が存在しない場合
1. 「**Add a variable**」をクリック
2. **Key**: `ADMIN_PASSWORD`
3. **Value**: 新しいパスワードを入力（例: `Kyushoku2024!Secure`）
4. **Scopes**: すべてにチェック（Production, Deploy previews, Branch deploys）
5. 「**Save**」をクリック

### ステップ4: 再デプロイ
環境変数を変更した後、再デプロイが必要です：

1. 左サイドバーの「**Deploys**」タブをクリック
2. 「**Trigger deploy**」をクリック
3. 「**Clear cache and deploy site**」を選択
4. 再デプロイが開始されます（2-3分待つ）

### ステップ5: 新しいパスワードでログイン
1. https://kyushoku.site/admin/upload にアクセス
2. 設定した新しいパスワードを入力
3. CSVファイルをアップロード

## 推奨パスワード例
- `Kyushoku2024!Secure`
- `TsukubaMirai2024!`
- `Admin@Kyushoku2024`

**重要**: 強力なパスワードを使用してください（大文字、小文字、数字、記号を含む）

## トラブルシューティング

### 再デプロイ後も認証に失敗する場合
1. ブラウザのキャッシュをクリア
2. シークレットモードで試す
3. Netlifyの環境変数が正しく保存されているか再確認
4. 再デプロイが完了しているか確認（Deploysタブで確認）

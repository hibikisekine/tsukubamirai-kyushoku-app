# 🚀 クイックスタートガイド

## 5分でデプロイ完了！

### ステップ1: Supabaseのセットアップ（3分）

1. **Supabaseアカウント作成**
   - https://supabase.com にアクセス
   - 「Start your project」→ GitHubでログイン

2. **プロジェクト作成**
   - 「New Project」をクリック
   - プロジェクト名: `tsukubamirai-kyushoku`
   - データベースパスワード: 強力なパスワードを設定（**必ずメモ！**）
   - リージョン: `Northeast Asia (Tokyo)`
   - プラン: Free

3. **テーブル作成**
   - 左メニュー「SQL Editor」を開く
   - 「New query」をクリック
   - `supabase-setup.sql` の内容をコピー＆ペースト
   - 「Run」をクリック

4. **APIキー取得**
   - 左メニュー「Settings」→「API」
   - 以下の3つをコピー：
     - **Project URL**
     - **anon public key**
     - **service_role key**（秘密に！）

### ステップ2: GitHubリポジトリ作成（1分）

1. **GitHubでリポジトリ作成**
   - https://github.com にログイン
   - 右上「+」→「New repository」
   - リポジトリ名: `tsukubamirai-kyushoku-app`（任意）
   - 「Create repository」をクリック

2. **ローカルをGitHubにプッシュ**
   ```bash
   cd 給食サイト
   git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git
   git branch -M main
   git push -u origin main
   ```
   （YOUR_USERNAMEとREPO_NAMEを置き換えてください）

### ステップ3: Vercelデプロイ（1分）

1. **Vercelにログイン**
   - https://vercel.com にアクセス
   - GitHubアカウントでログイン

2. **プロジェクトインポート**
   - 「Add New...」→「Project」
   - 作成したGitHubリポジトリを選択
   - 「Import」をクリック

3. **環境変数設定**
   「Environment Variables」で以下を追加：

   | 変数名 | 値 |
   |--------|-----|
   | `NEXT_PUBLIC_SUPABASE_URL` | SupabaseのProject URL |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabaseのanon key |
   | `SUPABASE_SERVICE_ROLE_KEY` | Supabaseのservice_role key |
   | `ADMIN_PASSWORD` | 強力なパスワード |

4. **デプロイ実行**
   - 「Deploy」をクリック
   - 2-3分待つ

### ステップ4: 動作確認

1. **サイトにアクセス**
   - `https://your-project.vercel.app` にアクセス

2. **データアップロード**
   - `/admin/upload` にアクセス
   - 管理者パスワードを入力
   - `data/sample.csv` をアップロード

3. **完了！** 🎉

## トラブルシューティング

### ビルドエラー
- 環境変数が正しく設定されているか確認
- Vercelのログを確認

### データが保存されない
- SupabaseのAPIキーが正しいか確認
- SupabaseのTable Editorでテーブルが作成されているか確認

# ✅ Supabaseセットアップ完了！次のステップ

## ✅ 完了したこと

- Supabaseテーブル作成成功

## ステップ1: テーブルの確認（30秒）

1. Supabaseダッシュボードの左メニューから「Table Editor」をクリック
2. `kondate`テーブルが表示されていれば成功
3. テーブルが空（データなし）でも問題ありません

## ステップ2: APIキーの取得（1分）

1. 左メニューの「Settings」（歯車アイコン）をクリック
2. 「API」をクリック
3. 以下の3つをコピーしてメモ：

   ### Project URL
   - 例: `https://xxxxx.supabase.co`
   - 「Project URL」の下に表示

   ### anon public key
   - 「Project API keys」セクションの「anon」の「public」キー
   - 「Reveal」をクリックして表示
   - 長い文字列（`eyJhbGci...`で始まる）

   ### service_role key（重要：秘密に！）
   - 「Project API keys」セクションの「service_role」の「secret」キー
   - 「Reveal」をクリックして表示
   - 長い文字列（`eyJhbGci...`で始まる）
   - **このキーは絶対に公開しないでください！**

## ステップ3: GitHubにプッシュ（認証が必要）

ターミナルで以下を実行：

```bash
cd 給食サイト
git push -u origin main
```

認証情報を求められたら：
- ユーザー名: `hibikisekine`
- パスワード: GitHubのパスワードまたはPersonal Access Token

## ステップ4: Vercelでデプロイ（3分）

### 4-1. Vercelにログイン

1. https://vercel.com にアクセス
2. GitHubアカウントでログイン

### 4-2. プロジェクトをインポート

1. 「Add New...」→「Project」をクリック
2. `hibikisekine/tsukubamirai-kyushoku-app` を選択
3. 「Import」をクリック

### 4-3. 環境変数を設定

「Environment Variables」で以下を追加：

| 変数名 | 値 | 説明 |
|--------|-----|------|
| `NEXT_PUBLIC_SUPABASE_URL` | SupabaseのProject URL | ステップ2で取得 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabaseのanon key | ステップ2で取得 |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabaseのservice_role key | ステップ2で取得 |
| `ADMIN_PASSWORD` | 強力なパスワード | CSVアップロード用 |

### 4-4. デプロイ実行

1. 「Deploy」ボタンをクリック
2. 2-3分待つ

### 4-5. デプロイ後の確認

1. デプロイが完了すると、URLが表示されます
   - 例: `https://tsukubamirai-kyushoku-app.vercel.app`
2. サイトにアクセスして動作確認
3. `/admin/upload` にアクセスしてCSVアップロードをテスト

これでアプリがWeb上で公開されました！🎉

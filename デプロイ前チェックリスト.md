# ✅ デプロイ前チェックリスト

## 必須項目

- [ ] `package.json`が正しく設定されている
- [ ] `.gitignore`に機密情報が含まれていない
- [ ] 環境変数`.env.local`が`.gitignore`に含まれている
- [ ] `ADMIN_PASSWORD`を強力なパスワードに変更する準備ができている
- [ ] ビルドが正常に完了する（`npm run build`が成功する）

## 推奨項目

- [ ] サンプルデータをアップロードして動作確認
- [ ] モバイル表示を確認
- [ ] すべてのページが正常に表示されることを確認
- [ ] Google AdSenseの設定（使用する場合）

## デプロイ前のテスト

### ローカルでビルドテスト

```bash
cd 給食サイト
npm run build
npm start
```

`http://localhost:3000`で動作確認してください。

### 確認すべきページ

- [ ] `/` - トップページ
- [ ] `/calendar` - カレンダーページ
- [ ] `/search` - 検索ページ
- [ ] `/about` - このアプリについて
- [ ] `/admin` - 管理者ダッシュボード
- [ ] `/admin/upload` - CSVアップロードページ

## 注意事項

⚠️ **重要**: 現在の実装では、データは`data/kondate.json`ファイルに保存されます。Vercelのサーバーレス環境では、このファイルは一時的なストレージに保存されるため、再デプロイ時にデータがリセットされる可能性があります。

本番環境でデータを永続化するには、データベースの導入を推奨します。

## 残りの手作業（最小限）

### 1. Supabaseでプロジェクト作成（2分）

1. https://supabase.com/dashboard/org/ypnwwsifgccmhyldkbhx にアクセス
2. 「New project」をクリック
3. プロジェクト名: `tsukubamirai-kyushoku`
4. パスワード設定（メモ）
5. リージョン: `Northeast Asia (Tokyo)`
6. 「Create new project」をクリック

### 2. SQL実行（1分）

1. プロジェクトが作成されたら、上部の検索バーで「SQL」と検索
2. 「SQL Editor」を開く
3. `supabase-setup.sql` の内容をコピー＆ペースト
4. 「Run」をクリック

### 3. APIキー取得（1分）

1. 「Settings」→「API」
2. 3つのキーをコピー：
   - Project URL
   - anon public key
   - service_role key

### 4. GitHubにプッシュ（認証が必要）

ターミナルで：
```bash
cd 給食サイト
git push -u origin main
```
（認証情報を入力）

### 5. Vercelでデプロイ（3分）

1. https://vercel.com にログイン
2. 「Add New...」→「Project」
3. GitHubリポジトリを選択
4. 環境変数を設定（SupabaseのAPIキー）
5. 「Deploy」をクリック

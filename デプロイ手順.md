# 🚀 デプロイ手順

## Vercelへのデプロイ（推奨）

### 1. GitHubリポジトリの作成

```bash
cd 給食サイト
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/your-username/tsukubamirai-kyushoku-app.git
git push -u origin main
```

### 2. Vercelでのデプロイ

1. [Vercel](https://vercel.com)にアクセスしてアカウント作成
2. 「New Project」をクリック
3. GitHubリポジトリを選択
4. プロジェクト設定：
   - **Framework Preset**: Next.js
   - **Root Directory**: `./給食サイト`（またはプロジェクトルート）
   - **Build Command**: `npm run build`
   - **Output Directory**: `.next`

### 3. 環境変数の設定

Vercelのプロジェクト設定で環境変数を追加：

```
ADMIN_PASSWORD=your-secure-password-here
NEXT_PUBLIC_ADSENSE_CLIENT_ID=ca-pub-xxxxxxxxxxxxx
```

### 4. デプロイ実行

- 自動デプロイ: GitHubにプッシュすると自動デプロイ
- 手動デプロイ: Vercelダッシュボードから「Deploy」をクリック

## その他のデプロイ方法

### Netlify

1. [Netlify](https://www.netlify.com)にアカウント作成
2. GitHubリポジトリを接続
3. ビルド設定：
   - Build command: `npm run build`
   - Publish directory: `.next`
4. 環境変数を設定

### Railway

1. [Railway](https://railway.app)にアカウント作成
2. GitHubリポジトリを接続
3. 環境変数を設定
4. 自動デプロイが開始

### 自前サーバー

```bash
# ビルド
npm run build

# 本番サーバー起動
npm start
```

## デプロイ後の確認事項

- [ ] サイトが正常に表示される
- [ ] CSVアップロード機能が動作する
- [ ] 献立データが正しく表示される
- [ ] モバイル表示が正常
- [ ] PWA機能が動作する（ホーム画面に追加可能）
- [ ] Google AdSenseが表示される（設定済みの場合）

## トラブルシューティング

### ビルドエラー

- Node.jsのバージョンを確認（18以上推奨）
- 依存関係が正しくインストールされているか確認

### 環境変数が反映されない

- Vercelの環境変数設定を確認
- 再デプロイを実行

### PWAが動作しない

- HTTPSでアクセスしているか確認
- `manifest.json`が正しく配置されているか確認
